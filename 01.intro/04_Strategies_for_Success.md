# Strategies for Success (성공 전략)

단순히 "만들어줘"라고 지시하는 것은 하수입니다. 시니어 개발자는 에이전트를 **전략적**으로 리딩해야 합니다.

## 1. Meta-Prompting & Architecture Design
코딩을 바로 시키지 말고, **설계**를 먼저 시키십시오.

- **Bad**: "로그인 기능 만들어줘."
- **Good**: "로그인 기능을 구현하기 위한 기술 명세서와 아키텍처 설계를 작성해줘. 데이터 흐름, 에러 처리 전략, 사용할 라이브러리 목록을 포함해."
- **효과**: `Implementation Plan` Artifact를 통해 방향성을 먼저 합의하고 코딩에 들어감으로써, 재작업 비용을 최소화합니다.

## 2. Multi-Agent Orchestration (멀티 에이전트 오케스트레이션)
여러 에이전트를 생성하여 팀처럼 운용하십시오.

- **Backend Agent**: "DB 스키마 설계 및 API 구현해."
- **Frontend Agent**: "구현된 API 명세를 보고 UI 컴포넌트 개발해."
- **QA Agent**: "브라우저 열어서 기능 테스트하고 버그 리포트 써줘."
- **효과**: 각 에이전트가 좁은 범위(Context)에 집중하게 하여 환각을 줄이고 전문성을 높입니다.

## 3. Cross-Model Verification (교차 모델 검증)
단일 모델(Gemini)에만 의존하지 말고, 다른 모델을 심사위원으로 활용하십시오.

- **프로세스**:
  1. Antigravity가 코드 작성.
  2. 코드를 복사해서 Claude 3.5 Sonnet이나 GPT-4o에게 입력.
  3. 프롬프트: "이 코드는 제미나이가 짠 거야. 보안 취약점이나 로직 오류를 비판적으로 까줘(Critique)."
  4. 피드백을 다시 Antigravity에게 전달하여 수정.
- **효과**: AI끼리의 동료 리뷰(Peer Review) 시스템 구축.

## 4. Agentic TDD (테스트 주도 개발)
테스트 코드는 AI에게 가장 명확한 '목표'이자 '가이드레일'입니다.

- **전략**: "이 기능을 검증할 수 있는 실패하는 테스트 케이스를 먼저 작성해. 그리고 그 테스트를 통과할 때까지 코드를 수정해."
- **효과**: 에이전트가 "테스트 통과"라는 명확한 종료 조건을 갖게 되어, 스스로 반복(Iterate)하며 완성도를 높입니다.
