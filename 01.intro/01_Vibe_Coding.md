# Vibe Coding (바이브 코딩)

## 정의
**바이브 코딩(Vibe Coding)**은 2025년 Andrej Karpathy가 주창한 개념으로, 개발자가 엄격한 구문(Syntax)이나 메모리 관리에 천착하기보다는, **자연어**를 통해 AI에게 원하는 결과물의 **'의도(Intent)'와 '느낌(Vibe)'**을 전달하여 코드를 생성하는 새로운 개발 패러다임을 의미합니다.

> "Forget the code exists. Focus on the intent."

## 핵심 철학

### 1. From Syntax to Intent
전통적인 코딩이 "어떻게(How)"를 기술하는 과정이었다면, 바이브 코딩은 **"무엇을(What)"과 "왜(Why)"**에 집중하는 선언적 프로그래밍의 확장된 형태입니다. 
- **기존**: `for` 루프를 어떻게 돌릴지, 변수 타입을 무엇으로 할지 고민.
- **바이브 코딩**: 데이터 흐름의 목적과 최종 사용자 경험(UX)을 설명.

### 2. Code First, Refine Later
AI 에이전트를 통해 일단 돌아가는 코드를 빠르게 생성하고(Prototyping), 이후에 세부적인 수정과 최적화를 진행합니다. 이는 아이디어를 즉시 실행 가능한 소프트웨어로 변환하는 속도를 극적으로 높여줍니다.

### 3. Developer as Orchestrator
개발자는 더 이상 '코더(Coder)'가 아닙니다. AI 에이전트 군단을 지휘하는 **지휘자(Orchestrator)**이자 **아키텍트(Architect)**로서, 전체 시스템의 설계와 품질 검증에 집중해야 합니다.

## Antigravity와의 관계
Google Antigravity는 이러한 바이브 코딩을 실현하기 위한 최적의 플랫폼입니다. 단순한 자동 완성을 넘어, 개발자가 '미션 컨트롤' 센터에서 에이전트들에게 "이 기능을 구현해", "이 버그를 잡아"라고 지시하면, 에이전트들이 실제 코딩 작업을 수행합니다.
