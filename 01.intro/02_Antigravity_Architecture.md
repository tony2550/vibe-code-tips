# Google Antigravity 아키텍처

Antigravity는 인간 개발자, AI 모델(Gemini 3 Pro), 그리고 실행 환경이 유기적으로 결합된 복합 시스템입니다. 이를 효과적으로 제어하기 위해서는 플랫폼의 구조와 산출물 시스템을 이해해야 합니다.

## 3-Surface Architecture
Antigravity는 개발 환경을 세 가지 표면(Surface)으로 확장하여 제공합니다.

### 1. Agent Manager (에이전트 매니저)
- **역할**: 비동기 작업의 허브이자 미션 컨트롤 센터.
- **기능**:
  - 여러 에이전트를 동시에 생성(Spawn)하여 병렬 작업 수행 (예: 백엔드 API 수정 + 프론트엔드 UI 변경).
  - 전체 작업 흐름을 모니터링하고 에이전트 간의 조율 담당.
  - 단일 스레드 개발자를 **멀티 스레드 관리자**로 전환.

### 2. Editor (에디터)
- **기반**: VS Code 포크(Fork).
- **특징**: 
  - 친숙한 코딩 환경이지만, 인간과 에이전트가 공존하는 공간.
  - 에이전트가 직접 파일을 생성, 수정, 삭제하는 과정을 실시간으로 감독(Supervise).
  - **인라인 명령어(Cmd+I)**: 코드 중간에 개입하여 에이전트에게 즉각적인 수정 지시 가능.

### 3. Browser (브라우저)
- **특징**: Headless가 아닌 실제 GUI를 가진 내장(embedded) 크롬 인스턴스.
- **기능**:
  - **Research**: 에이전트가 웹 문서를 검색하여 최신 기술 정보를 습득.
  - **Validation**: 로컬 호스트(localhost)에 접속하여 실제 버튼을 클릭하고 UI 렌더링을 시각적으로 검증.
  - 프론트엔드 개발 시 인간의 개입을 최소화하는 결정적 요소.

## 산출물 (Artifacts) 시스템
안티그래비티의 핵심은 에이전트의 작업을 투명하게 만들고 검증 가능하게 하는 '산출물'입니다.

### 1. 작업 목록 (Task List) `task.md`
- **목적**: 에이전트가 이해한 요구사항을 실행 가능한 단위로 분해.
- **활용**: 전체 진행 상황을 트래킹하고, 누락된 요구사항이 없는지 초기에 점검.

### 2. 구현 계획 (Implementation Plan) `implementation_plan.md`
- **목적**: **코드를 건드리기 전**에 아키텍처와 로직을 제안하는 기술 명세서.
- **검증 포인트**:
  - 기존 아키텍처 위반 여부.
  - 보안 취약점 및 데이터 무결성 리스크.
  - **Action**: 마음에 들지 않으면 말로 하지 말고, 문서에 직접 코멘트를 달아 수정 지시 (Google Docs처럼).

### 3. 워크스루 (Walkthrough) `walkthrough.md`
- **목적**: 작업 완료 후의 결과 보고서.
- **내용**: 변경된 파일 목록, 수행한 테스트 결과, UI 스크린샷 등.
- **활용**: PR(Pull Request) 설명서로 활용하거나, 다음 작업을 위한 인수인계 문서로 사용.
