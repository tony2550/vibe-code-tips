# Risk Management (리스크 관리)

바이브 코딩은 생산성을 극대화하지만, 동시에 새로운 형태의 기술 비제들을 야기할 수 있습니다. 이를 인지하고 방어하는 것이 시니어 개발자의 역할입니다.

## 1. Vibe Coding Hangover (바이브 행오버)
- **증상**: "일단 돌아가니까 OK"라며 생성된 코드를 무비판적으로 수용하다가, 나중에 유지보수 불가능한 상태에 직면하는 현상 (마치 술 깬 뒤 찾아오는 숙취와 같음).
- **리스크**:
  - 중복 코드 양산 (DRY 원칙 위반).
  - 프로젝트 컨벤션을 무시한 이질적인 코드 스타일.
- **방어 전략**:
  - **Diff 확인 필수**: 아무리 작은 변경이라도 반드시 변경 내역을 확인하십시오.
  - **Refactoring 지시**: "작동은 하는데, 우리 프로젝트 유틸리티 함수를 써서 코드를 더 깔끔하게 정리해줘"라고 추가 지시를 내려야 합니다.

## 2. Hallucination & Logic Errors (환각과 논리 오류)
- **증상**: 구문(Syntax)은 완벽하지만, 비즈니스 로직(Logic)이 틀린 코드를 생성.
- **예시**:
  - 예외 처리를 누락하고 단순히 에러를 삼키는(`swallow exception`) 코드.
  - DB 트랜잭션 롤백 로직 누락.
- **방어 전략**:
  - **핵심 로직 검증**: 결제, 데이터 저장 등 중요한 로직은 반드시 인간이 직접 라인 단위로 검증해야 합니다.
  - **TDD 강제**: "엣지 케이스를 포함한 테스트 코드를 먼저 짜줘"라고 하여 AI가 스스로 검증하게 만듭니다.

## 3. Security Threats (보안 위협)
- **증상**: 에이전트가 터미널 권한을 이용해 위험한 행동을 함.
- **예시**:
  - `npm install` 중 악성 패키지 설치 (Typosquatting).
  - DB 초기화(`DROP DATABASE`)나 리소스 삭제(`rm -rf`) 명령어 실행.
  - 클라우드 인증 정보 노출.
- **방어 전략**:
  - **터미널 정책 강화**: `settings.json`에서 `executionPolicy`를 'auto' 대신 'review required'로 설정하거나, 화이트리스트를 운용하십시오.

## 4. Context Drift (컨텍스트 표류) & Loop of Death
- **증상**: 대화가 길어질수록 초기 지시사항을 잊거나, 특정 에러를 고치려다 무한 루프에 빠져 코드를 망가뜨림.
- **방어 전략**:
  - **Session Chaptering**: 하나의 세션에서 모든 걸 해결하려 하지 마십시오. 기능 단위로 세션을 끊고(Reset), 요약본을 들고 새 방으로 이동하십시오.
  - **Health Check**: 주기적으로 에이전트에게 "현재 목표가 뭐지?"라고 물어 컨텍스트 상태를 점검하십시오.

## 5. Dependency Bloat (의존성 비대화)
- **증상**: 간단한 기능을 위해 무거운 외부 라이브러리를 무분별하게 설치.
- **방어 전략**: `package.json` 변경 사항을 항상 예의시하고, 불필요한 라이브러리 추가 시 반려하십시오.
