# 클로드 코드 창시자가 직접 알려주는 클로드 코드 꿀팁 13가지

## 1. 작업 효율 및 워크플로우 극대화 (Workflow)
보리스 체니는 '분신술'처럼 여러 세션을 동시에 운용하여 작업 속도를 비약적으로 높입니다.

병렬 실행 (Parallel Execution): 로컬에서 5개의 클로드 세션을 동시에 실행하며, 사용자 입력이 필요할 때 시스템 알림이 오도록 설정하여 즉시 반응합니다. [00:56]

웹 세션 활용: 로컬 세션 외에도 웹 세션 5~10개를 추가로 실행하여, 총 15개 정도의 세션을 동시에 활용합니다. 백그라운드 작업(&) 기능을 적극 활용합니다. [01:14]

장기 작업 처리 (Long-running Tasks): 작업이 완료되면 백그라운드 에이전트가 검증하도록 하고, 무한 반복 작업(iteration)을 위해 '윌건(Willgan) 플러그인' 등을 활용하여 효율성을 높입니다. [05:08]

## 2. 모델 선택 및 권한 설정 (Model & Configuration)
비용보다는 결과물의 품질과 사람의 개입 최소화를 최우선으로 합니다.

최고 성능 모델 사용: 무조건 Opus 4.5와 Thinking Mode를 사용합니다. 비용이 들고 속도가 느리더라도, 한 번에 정확한 결과를 내는 것이 사람의 수정 시간을 줄여 결국 더 빠르기 때문입니다. [01:41]

권한 관리 (Permissions): dangerously-skip-permissions 모드를 사용하지 않습니다. 대신 자주 쓰는 명령어는 /permissions 명령어로 미리 허용 목록(whitelist)에 등록하고, config.json에 저장해 팀과 공유합니다. [04:18]

도구 접근 권한 부여: 슬랙(Slack), 빅쿼리(BigQuery), 센트리(Sentry) 등 외부 도구 접근 권한을 주어 클로드가 직접 로그를 확인하고 분석하게 합니다. [04:49]

## 3. 지식 관리 및 팀 협업 (Knowledge & Teamwork)
클로드의 '기억'을 문서화하고 팀 전체가 공유하여 실수를 줄입니다.

CLAUDE.md 적극 활용: CLAUDE.md 파일은 Git에 커밋하여 팀과 공유합니다. 클로드가 실수하면 재발 방지를 위한 지침을 이 파일에 추가하여 '자가 치유' 시스템을 구축합니다. [02:08]

PR 검증과 피드백: PR 검증 시 클로드를 태그하여 학습한 내용을 CLAUDE.md에 업데이트하도록 합니다. GitHub 앱을 설치하여 활용하는 것을 추천합니다. [02:39]

커스텀 커맨드 공유: 반복되는 워크플로우는 커맨드로 저장하여 프롬프트 입력 시간을 줄이고, 이를 팀원들과 공유합니다. [03:22]

## 4. 품질 관리 및 자동화 (Quality Control & Automation)
계획 단계부터 검증까지 체계적인 프로세스를 따릅니다.

플랜 모드(Plan Mode) 시작: 작업 시작 시 바로 코딩하지 않고 '플랜 모드(Shift+Tab)'로 진입하여 계획을 먼저 세웁니다. 계획이 확정되면 실행 모드로 전환하여 한 번에 결과물을 만듭니다. [02:48]

서브 에이전트(Sub-agents) 활용: 코드 정리용(Simplifier), 테스트 검증용(Verifier) 등 특정 목적에 맞는 서브 에이전트를 정의하여 사용합니다. [03:40]

자동 포매팅 (Post-tool Hooks): 클로드가 코드를 작성한 후 자동으로 포매팅(Formatting)을 수행하도록 훅을 설정하여 사소한 오류를 방지합니다. [04:01]

자가 검증 (Self-Verification): 클로드에게 작업 후 스스로 검증할 수 있는 명확한 방법(테스트 코드 실행, 브라우저 테스트 등)을 제시하여 결과물의 신뢰도를 높입니다. [05:27]
